⚡ Palskills

AI-Powered Development Pipeline — 8 skills, one knowledge graph

🚀 What is Palskills?

An AI-native development methodology that breaks the software lifecycle into 8 specialized skills, each owning one phase. No skill overlaps. Every decision is captured in a knowledge graph (.palbox/).

🔍 Lyleen→ 🎨 Elphidran→ 🧩 Astegon→ 🏗️ Blazamut→ 📋 Jetdragon→ ⚔️ Anubis→ 📝 Panthalus→ 🎯 Astralym
🔥 Key insight: Each skill decides, not suggests. Lyleen discovers codebases. Astegon decides components. Blazamut designs modules. Anubis executes. Panthalus records. No more "what should I build?" — the skills tell you.

🧠 The Palbox — Knowledge Graph

Every decision, discovery, and session is stored as an Obsidian-style graph with [[wikilinks]].

.palbox/
  ├── state.md # Pipeline progress tracker
  ├── README.md # Root node → [[architecture]], [[methods]]
  ├── architecture.md # Codebase map
  ├── design.md # Design tokens (Elphidran)
  ├── methods.md # Conventions & standards
  ├── flows/ # Lyleen DISCOVER → auto-saved
  ├── components/ # Astegon specs
  ├── architectures/ # Blazamut specs
  ├── plans/ # Jetdragon plans
  └── history/ # Panthalus session records
🔗 Domain separation: Lyleen→flows/ | Astegon→components/ | Blazamut→architectures/ | Panthalus→history/
Never cross domains. Every edge is a [[wikilink]].
💎 Design Principles
  • ✓ Decide, don't suggest — Skill output is authoritative, not advisory
  • ✓ SRP Everywhere — One component, one job. One class, one reason to change.
  • ✓ Ponytail (Token Efficiency) — Lazy senior dev. Shortest diff wins. Boring > clever.
  • ✓ On-Demand Capture — Only what matters goes to palbox. No dumpster.
  • ✓ Cross-Agent — Works in Codex, Cursor, Claude Code. Same skills, same graph.
  • ✓ Knowledge Graph, not Docs — [[wikilinks]] over flat files. Bidirectional edges.
  • ✓ Resumable Pipeline — state.md checkboxes. Crash? Restart from last [ ].
